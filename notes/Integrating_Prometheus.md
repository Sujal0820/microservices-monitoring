# Step 4: Integrating Prometheus for Metrics Collection

## Overview

In this step, Prometheus was integrated into the system to collect and store metrics generated by the microservice.

This transforms the system from simply producing metrics to actually monitoring and analyzing them over time.

---

## Objective

* Deploy Prometheus using Docker
* Configure it to scrape metrics from the API Gateway
* Validate that metrics are being collected successfully

---

## Prometheus Configuration

A configuration file was created at:

```id="p1"
/monitoring/prometheus/prometheus.yml
```

### Configuration Details

* scrape_interval: 5 seconds
  → Defines how frequently metrics are collected

* job_name: api-gateway
  → Logical name of the monitored service

* metrics_path: /metrics
  → Endpoint exposed by the service

* targets: api-gateway:8000
  → Docker service name used for internal communication

---

## Docker Integration

Prometheus was added as a service in `docker-compose.yml`.

It was configured with:

* Official Prometheus image
* Volume mapping for configuration
* Port exposure (9090) for UI access

---

## System Flow After Integration

```id="p2"
FastAPI Service → /metrics → Prometheus (scraping every 5s)
```

Prometheus continuously collects and stores time-series data.

---

## Verification Steps

### 1. Access Prometheus UI

```id="p3"
http://localhost:9090
```

---

### 2. Check Targets

Navigate to:
Status → Targets

Expected:

* api-gateway → UP

---

### 3. Run Query

```id="p4"
http_requests_total
```

This confirms that metrics are being successfully ingested.

---

## Key Learning

* Prometheus works on a pull-based model (scraping)
* Service discovery inside Docker relies on container names
* Metrics must be exposed in a compatible format for Prometheus

---

## Challenges Faced

* Understanding how Docker networking resolves service names
* Ensuring correct target configuration
* Validating metrics availability before querying

---

## Outcome

A fully functional monitoring pipeline where:

* Metrics are generated by the service
* Prometheus collects and stores them
* Data is ready for visualization
